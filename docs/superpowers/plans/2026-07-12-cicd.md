# CI/CD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** CI checks on PRs/pushes to main, tag-driven GitHub Releases (macOS Apple Silicon + Windows) via tauri-action, in-app auto-updates, and a `curl | bash` macOS installer.

**Architecture:** Two GitHub Actions workflows — `ci.yml` (Biome + frontend build + `cargo check`, parallel Linux jobs) and `release.yml` (tag-guard job, then a two-platform matrix running `tauri-apps/tauri-action` into a shared draft release with updater artifacts and `latest.json`). The app gains `tauri-plugin-updater` (check on launch → sonner toast → download/install/relaunch). `install.sh` at the repo root downloads the `.app.tar.gz` from the latest release via curl (which never sets the quarantine attribute, so Gatekeeper's "damaged" dialog never fires).

**Tech Stack:** GitHub Actions, `tauri-apps/tauri-action@v0`, `pnpm/action-setup@v4`, `actions/setup-node@v4`, `dtolnay/rust-toolchain@stable`, `swatinem/rust-cache@v2`, `tauri-plugin-updater` v2, bash.

**Spec:** `docs/superpowers/specs/2026-07-12-cicd-design.md`

## Global Constraints

- Node **24** in all workflows; pnpm version comes from `packageManager: "pnpm@10.15.0"` in package.json (never pass a version to `pnpm/action-setup`).
- macOS builds target **`aarch64-apple-darwin`** only; Windows builds **NSIS only** (no MSI) — enforced via `--bundles` CLI args in the release matrix, NOT by narrowing `bundle.targets` in tauri.conf.json (dmg/nsis are host-specific; local `pnpm package` must keep working).
- Releases are **drafts**; updater endpoint is exactly `https://github.com/tasjen/daily-report/releases/latest/download/latest.json`.
- No OS code signing: `bundle.macOS.signingIdentity` stays `"-"`.
- Secret names: `TAURI_SIGNING_PRIVATE_KEY`, `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` (updater artifact signing — not OS code signing).
- **No `gh` CLI on this machine** — GitHub interactions go through the web UI or unauthenticated `curl` to `api.github.com`.
- Commit messages: imperative sentence-case matching repo history (e.g. "Add CI workflow"), each ending with the line `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
- Biome runs on pre-commit via lefthook and will auto-fix staged files; if a commit is amended by hooks, keep going.
- Work happens on the existing `feat/cicd` branch.

```yaml
name: CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: ${{ github.event_name == 'pull_request' }}

jobs:
  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: pnpm
      - run: pnpm install --frozen-lockfile

      - run: pnpm build

    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

          sudo apt-get update

      - uses: dtolnay/rust-toolchain@stable
      - uses: swatinem/rust-cache@v2
        with:
          workspaces: src-tauri
      - run: cargo check --manifest-path src-tauri/Cargo.toml --locked
```

- [ ] **Step 1: Add the npm package**

Run: `pnpm add @tauri-apps/plugin-updater`
Expected: package.json gains `"@tauri-apps/plugin-updater": "^2..."` and the lockfile updates.

- [ ] **Step 2: Write the hook**

Create `src/lib/use-update-check.ts`:

```ts
import { relaunch } from "@tauri-apps/plugin-process";
import { check } from "@tauri-apps/plugin-updater";
import { useEffect } from "react";
import { toast } from "sonner";

export function useUpdateCheck() {
  useEffect(() => {
    if (import.meta.env.DEV) {
      return;
    }
    check()
      .then((update) => {
        if (!update) {
          return;
        }
        toast.info(`Update available: v${update.version}`, {
          duration: Number.POSITIVE_INFINITY,
          action: {
            label: "Update & restart",
            onClick: () => {
              // On Windows the NSIS installer exits the app itself before
              // relaunch() is reached; the relaunch matters on macOS.
              toast.promise(update.downloadAndInstall().then(relaunch), {
                loading: "Downloading update…",
                success: "Restarting…",
                error: (e) => `Update failed: ${e}`,
              });
            },
          },
        });
      })
      .catch(() => {
        // Offline or the release endpoint is unreachable — a launch-time
        // update check is not worth surfacing errors for.
      });
  }, []);
}
```

In `src/App.tsx`: add the import (Biome will sort it):

```ts

```

and inside `App()`, directly after `useResetWhenAway();`, add:

### Task 6: Release workflow

**Files:**
- Create: `.github/workflows/release.yml`
- Modify: `docs/superpowers/specs/2026-07-12-cicd-design.md` (one bullet — see Step 2)

**Interfaces:**
- Consumes: secrets from Task 3; `createUpdaterArtifacts` config from Task 4.
- Produces: on `v*` tag push — a draft GitHub Release containing the `.dmg`, the NSIS `-setup.exe`, updater artifacts + `.sig` files, and `latest.json`. `install.sh` (Task 7) depends on the published release exposing an asset ending in `.app.tar.gz`.

- [ ] **Step 1: Write the workflow**

Create `.github/workflows/release.yml`:

```yaml
name: Release

on:
  push:
    tags:
      - "v*"

permissions:
  contents: write

jobs:
  guard:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Verify tag matches tauri.conf.json version
        run: |
          TAG_VERSION="${GITHUB_REF_NAME#v}"
          CONF_VERSION="$(jq -r .version src-tauri/tauri.conf.json)"
          if [ "$TAG_VERSION" != "$CONF_VERSION" ]; then
            echo "Tag v${TAG_VERSION} does not match tauri.conf.json version ${CONF_VERSION}" >&2
            exit 1
          fi

  build:
    needs: guard
    strategy:
      fail-fast: false
      matrix:
        include:
          - platform: macos-latest
            rust-targets: aarch64-apple-darwin
            args: --target aarch64-apple-darwin --bundles dmg
          - platform: windows-latest
            rust-targets: x86_64-pc-windows-msvc
            args: --bundles nsis
    runs-on: ${{ matrix.platform }}
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: pnpm
      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: ${{ matrix.rust-targets }}
      - uses: swatinem/rust-cache@v2
        with:
          workspaces: src-tauri
      - run: pnpm install --frozen-lockfile
      - uses: tauri-apps/tauri-action@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
          TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
        with:
          tagName: v__VERSION__
          releaseName: "Daily Report v__VERSION__"
          releaseDraft: true
          prerelease: false
          args: ${{ matrix.args }}
```

(`v__VERSION__` is a tauri-action literal — the action substitutes the app version; both matrix legs resolve to the same tag and share one draft release. The updater artifacts and `latest.json` are included automatically because `createUpdaterArtifacts` is enabled and the signing env vars are present.)

### Task 7: macOS install script

**Files:**
- Create: `install.sh` (repo root)

**Interfaces:**
- Consumes: a *published* GitHub release with an asset ending in `.app.tar.gz` (produced by Task 6; none exists until the first tag after merge — the script's release lookup will 404 until then, which is expected).
- Produces: the one-liner documented in Task 8's README: `curl -fsSL https://raw.githubusercontent.com/tasjen/daily-report/main/install.sh | bash`.

- [ ] **Step 1: Write the script**

Create `install.sh`:

```bash
#!/bin/bash
set -euo pipefail

REPO="tasjen/daily-report"
APP_NAME="Daily Report"
APP_PATH="/Applications/$APP_NAME.app"

if [ "$(uname -s)" != "Darwin" ]; then
  echo "This installer is for macOS only. Windows installers: https://github.com/$REPO/releases/latest" >&2
  exit 1
fi

if [ "$(uname -m)" != "arm64" ]; then
  echo "Only Apple Silicon (arm64) builds are available; this Mac is $(uname -m)." >&2
  exit 1
fi

echo "Looking up the latest release..."
ASSET_URL="$(curl -fsSL "https://api.github.com/repos/$REPO/releases/latest" |
  grep -o '"browser_download_url": *"[^"]*\.app\.tar\.gz"' |
  head -n 1 | cut -d '"' -f 4)"

if [ -z "$ASSET_URL" ]; then
  echo "Could not find a .app.tar.gz asset in the latest release." >&2
  exit 1
fi

TMP_DIR="$(mktemp -d)"
trap 'rm -rf "$TMP_DIR"' EXIT

echo "Downloading $ASSET_URL"
curl -fL --progress-bar "$ASSET_URL" -o "$TMP_DIR/app.tar.gz"

echo "Installing to $APP_PATH"
osascript -e "quit app \"$APP_NAME\"" 2>/dev/null || true
rm -rf "$APP_PATH"
tar -xzf "$TMP_DIR/app.tar.gz" -C /Applications

# curl downloads carry no quarantine attribute; this is belt-and-braces in
# case the tarball was produced from a quarantined source.
xattr -dr com.apple.quarantine "$APP_PATH" 2>/dev/null || true

echo "Installed. Launching..."
open "$APP_PATH"
```
